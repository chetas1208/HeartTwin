# HeartTwin Lab — Synthetic Test Fixtures

> **Educational cardiac simulation only. Not for diagnosis or treatment decisions.**

## What this is

Every file in this directory is **synthetic** test data generated by
[`scripts/create_synthetic_fixtures.py`](../../scripts/create_synthetic_fixtures.py).

- **All data is synthetic.** No values were taken from a real patient.
- **Inspired by public dataset structure only.** Field names and shapes are
  modeled on the *structure* of public datasets (EchoNet-Dynamic, PTB-XL,
  CAMUS, ACDC) — not their contents. Nothing here was downloaded from those
  datasets.
- **Not patient data.** Contains no PHI.
- **Not for diagnosis.** These fixtures exist only to test software behavior.

## Regenerating

The fixtures are committed for convenience, but they are fully reproducible:

```bash
python scripts/create_synthetic_fixtures.py
```

The script is idempotent — it only rewrites a file when its content changes.

## Files

| File | Purpose |
|---|---|
| `manual_baseline.json` | Baseline manual vitals + expected deterministic outputs (SV 70, EF 58.33, CO 5.04, MAP 93.33, RR 833.33) |
| `manual_reduced_function.json` | Reduced pump function simulation input (SV 55, EF 36.67, CO 4.84, MAP 101.67, RR 681.82) |
| `manual_partial_data.json` | Partial data stress test (EDV/ESV missing) — must not crash, must warn, must not invent SV |
| `report_baseline.txt` | Synthetic report text for PDF/text extraction path |
| `report_reduced_function.txt` | Synthetic reduced-function report text |
| `report_partial_data.txt` | Synthetic report text with missing EDV/ESV |
| `ecg_synthetic_normal.csv` | ~10 s synthetic ECG-like waveform (`time_ms,lead_ii_mv`) at ~72 bpm |
| `ecg_synthetic_fast.csv` | Faster-HR synthetic ECG-like waveform at ~120 bpm |
| `echo_metadata_baseline.json` | Synthetic echo metadata (no video), baseline EDV/ESV/EF |
| `echo_metadata_reduced_function.json` | Synthetic echo metadata, reduced function |
| `vista3d_success_response.json` | Synthetic VISTA-3D success response for client parsing tests |
| `vista3d_failure_response.json` | Synthetic VISTA-3D failure response for fail-safe tests |

## Labeling rule

`manual_reduced_function.json` and `report_reduced_function.txt` represent
**reduced pump function simulation input**. They are never labeled as a
diagnosis. The application treats reported clinical terms as *reported
descriptors*, never as conclusions.
